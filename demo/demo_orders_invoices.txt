ORDER STATUS LEGEND
- Draft: order created but not approved.
- Confirmed: order approved and waiting for fulfillment.
- Picking: warehouse team is selecting items.
- Packed: items packed and waiting for carrier pickup.
- Shipped: carrier has collected the package.
- Delivered: customer received the order.
- On Hold: order requires review before fulfillment.
- Cancelled: order closed without shipment.
- Partially Fulfilled: at least one line item was shipped, but remaining items are pending.

INVOICE PAYMENT STATUS LEGEND
- Unbilled: order has not yet produced an invoice.
- Open: invoice issued and awaiting payment.
- Paid: invoice fully paid.
- Partially Paid: payment received but balance remains.
- Overdue: invoice has passed the due date without full payment.
- Credit Pending: a credit memo is being prepared or reviewed.

WAREHOUSE AND FULFILLMENT NOTES
- Warehouse zone A stores paper, notebooks, desk organizers, and low-fragility consumables.
- Warehouse zone B stores printers, label rolls, toner, electronics accessories, and shipping-label equipment.
- Warehouse zone C stores chairs, desks, monitor arms, lighting, and large furniture cartons.
- Orders with status On Hold must not be moved to Picking until the assigned owner clears the hold note.
- Orders marked Partially Fulfilled should show which item group remains pending in the notes field.
- Demo carrier cutoff is 16:30 local time Monday through Friday. Orders packed after the cutoff ship the next business day.
- Fragile or high-value orders should include an inspection note before shipment.

CUSTOMER SUPPORT PLAYBOOK FOR ORDER QUESTIONS
- If a user asks whether an order has shipped, answer from the Order Status field and cite the matching order row.
- If a user asks whether an invoice is overdue, answer from the Invoice Status and Due Date fields.
- If a user asks why an order is on hold, cite the hold reason from the Notes field.
- If a user asks for a customer phone number, fallback: this demo ledger intentionally does not contain phone numbers.
- If a user asks for real payment instructions, fallback: this demo document contains fictional information only.
- If a user asks about a missing order not listed here, say the order was not found in the uploaded documents.
